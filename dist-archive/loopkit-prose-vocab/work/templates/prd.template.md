# PRD — <title>

> Copy this into `work/` to start (e.g. `work/prd.md`, or a `work/prd/` folder split by section).
> Each section should be grounded in notes from `knowledge/` and bounded by anything in
> `knowledge/golden/`. Mark a section `locked: true` once it's finalized.

## Problem & context
What's wrong, for whom, with evidence from the knowledge base.

## Plan / approach
The recommended solution and rollout.

## Technical implementation
The build approach — must respect any NFRs, API contracts, or constraints pinned in `golden/`.

## Governance evidence
One line per required sign-off, each with its source (ticket, RFC, date).
