# Open Knowledge Format (OKF) Specification v0.1

OKF is an open, human- and agent-friendly format for representing knowledge: the metadata,
context, and curated insight around data and systems. Concretely it is a **directory tree of
UTF-8 markdown files with YAML frontmatter** — a "Knowledge Bundle." Intentionally minimal:
no schema registry, no central authority, no required tooling. *"If you can `cat` a file, you
can read OKF; if you can `git clone` a repo, you can ship it."* It is **specified** (it pins
the small set of interoperability rules) rather than merely conventional.

# Key points
- **Bundle = directory tree of markdown files.** Distribution: git repo (recommended),
  tarball/zip, or a subdirectory of a larger repo.
- **File = Concept.** Every non-reserved `.md` file is one unit of knowledge. **Concept ID =
  file path minus `.md`** (`tables/users.md` → `tables/users`). No UUIDs; identity is the path.
- **Frontmatter schema.** REQUIRED: `type` (non-empty string; producer-chosen, not registered;
  consumers MUST tolerate unknown values). RECOMMENDED (priority order): `title`,
  `description` (one sentence), `resource` (URI of the underlying asset; absent for abstract
  concepts), `tags` (YAML list), `timestamp` (ISO 8601). EXTENSIONS: any extra keys allowed;
  consumers SHOULD preserve unknown keys on round-trip.
- **Reserved filenames:** `index.md` (directory listing / progressive disclosure) and
  `log.md` (chronological update history). Never used as concept docs. `index.md` carries no
  frontmatter — *except* the bundle-root `index.md`, the only place `okf_version` is declared.
- **Links = untyped directed edges.** Standard markdown links; relationship kind conveyed by
  surrounding prose. Bundle-relative absolute links (leading `/`) are RECOMMENDED (stable
  under moves). Consumers **MUST tolerate broken links** (= not-yet-written knowledge).
- **Body conventions:** favor structural markdown; conventional headings `# Schema`,
  `# Examples`, `# Citations` (numbered `[1] [text](url)`).
- **Conformance (v0.1):** every non-reserved `.md` has parseable frontmatter with a non-empty
  `type`; reserved files follow their sections when present.
- **Versioning:** `<major>.<minor>` — minor = backward-compatible additions; major = breaking
  changes.

# Notable excerpts
- "The format is intentionally minimal: a directory of markdown files with YAML frontmatter.
  There is no schema registry, no central authority, and no required tooling."
- "Type values are not registered centrally… consumers MUST tolerate unknown types gracefully."
- "Consumers MUST tolerate broken links — a link whose target does not exist… may simply
  represent not-yet-written knowledge."
- "This permissive consumption model is intentional: OKF is meant to remain useful as bundles
  grow, get refactored, and are partially generated by agents."
